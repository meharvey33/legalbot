# COMMERCIAL LEGAL FAQ

Welcome to the Commercial Legal FAQs.

Here you will find answers to the most commonly asked commercial legal questions.

If you cannot find the answer you need here then please DM your assigned attorney from the Commercial Legal team or post your question in the #legal slack channel. If you have a more in-depth customer-related question, please open a new issue in the Commercial Legal repo and an attorney will be assigned to assist you.

This FAQ was last updated in December 2025.

## NDA Questions

### Do we have a GitHub NDA?

No. We do not have a GitHub specific NDA. As GitHub is wholly owned by the Microsoft Corporation, GitHub is covered under the standard Microsoft NDA which extends equal protection to affiliates or subsidiaries of Microsoft, including GitHub.

If a customer already has a standard evergreen NDA in place with Microsoft, then this will cover GitHub as an affiliate of Microsoft.

### How do I know if a customer already has an NDA in place with Microsoft?

Find out how to check if there is a Microsoft NDA already in place here: https://github.com/github/CELA/blob/main/Non-Disclosure%20Agreements%20(NDA).md

### If there is no Microsoft NDA in place, how do I create an NDA?

If there is no Microsoft NDA in place, the customer will need to sign Microsoft's standard NDA.

Find out how to put an NDA in place here: https://github.com/github/CELA/blob/main/Non-Disclosure%20Agreements%20(NDA).md

### Will GitHub/Microsoft sign a customer's NDA?

Microsoft generally does not agree to customers' NDAs. The customer will need to agree to Microsoft's standard NDA.

If the customer insists on their NDA, you will need to escalate this request to Microsoft's Global Contracting Office: https://contracts.microsoft.com/

### Can a customer request changes to the Microsoft NDA?

Any requests to amend Microsoft's NDA must be reviewed by Microsoft's Global Contracting Office: https://contracts.microsoft.com/

## Contract Questions

### Can we agree to use a customer's contract?

No we cannot. GitHub's customer terms have been specifically tailored to GitHub's products and services. GitHub also operates on a one-to-many basis and it is not scalable to agree to bespoke customer terms. For these reasons we cannot agree to use a customer's contract and must use the GitHub Customer Agreement or the customer's existing Microsoft agreement if relevant.

### Where can I find the GitHub Customer Terms?

The GitHub Customer Terms are available online: https://github.com/customer-terms

### How do the GitHub Customer Terms work for GitHub direct customers?

Take a look at this Guru Card for more information on how the GitHub contract stack works for GitHub direct customers: https://app.getguru.com/card/cMkERgLi/The-GitHub-Customer-Agreement-GCA

### My customer has a Microsoft agreement, do they also need to sign the GitHub Customer Agreement?

No, if your customer is purchasing GitHub products via their Microsoft Agreement, they do not need to sign up to the GitHub Customer Agreement. Standard Microsoft agreements incorporate GitHub-specific terms via the Microsoft Product Terms for GitHub Offerings: https://www.microsoft.com/licensing/terms/productoffering/GitHubOfferings/allprograms

### What if a customer has questions about the GitHub legal terms?

Create an issue in the Commercial Legal Repository: https://github.com/github/commlegal/issues/new?template=01-commlegal-issue-general.yml
The designated lawyer for your region/business will respond to you in the issue.

### What if my GitHub direct customer wants to make changes to GitHub's customer terms?

The minimum threshold for GitHub direct customers asking to redline the GitHub legal terms is $100k ARR. If the customer meets this minimum threshold, they may redline only the GitHub General Terms and the GitHub Copilot Product Specific Terms. The other GitHub Product Specific Terms are non-negotiable.

For customers who have previously negotiated our terms, it is necessary to have an annual increase of at least $100k ARR for renewing customers who ask to renegotiate our commercial contracts.

There is a $500k minimum threshold for customers who wish to redline the GitHub Data Protection Agreement ("DPA") as DPA negotiations tend to be complex and time consuming across multiple teams.

Once you receive the customer redlines, upload the redlined terms to Ironclad.

### What if my Microsoft co-sell customer wants to negotiate their legal terms?

Please work with your Microsoft customer account counterpart to engage Microsoft Deal Desk. Customer requests to amend Microsoft paper are handled by Microsoft Deal Desk.

### My customer wants to change the governing law of the GitHub Customer Agreement.

If the customer meets the minimum $100k ARR threshold for negotiation then speak to GitHub Deal Desk about the customer's request to change the governing law.

Section 10.3 of the GitHub Customer Agreement enables GitHub Deal Desk to make changes to Section 10.11 (Governing Law and Venue) provided the governing law is one of those pre-approved by GitHub CELA. Typically this change, if approved, gets added as a line in an Order Form or Quote.

### My GitHub direct customer wants to add a PDF/static copy of the GitHub Customer Agreement to the Order Form.

We do not attach static copies of our terms to the Order Form. Instead, customers can download a copy of the terms in place at the date of the agreement from https://github.com/customer-terms. We have the contract version's month and year recorded on each agreement.

### My customer is concerned that GitHub will change the contract terms during the subscription term.

The customer can rest assured that we cannot change a customer's contracts unilaterally in the middle of a subscription term. The GitHub Customer Agreement General Terms are evergreen. The other terms (such as the Product Specific Terms or the GitHub Data Protection Agreement) will remain in effect, without updates or changes, during the customer's subscription term. Then, at the start of a new subscription, the customer will get the terms in effect at that time.

### Where can a customer see the changes that have been made to the GitHub legal terms?

All updates to the GitHub customer terms since December 2022 can be found here: https://github.com/customer-terms/updates

## Data Protection / DPA Questions

### What is the GitHub DPA and where can I find it?

The GitHub Data Protection Agreement is a legally binding contract that sets out the rights and obligations of GitHub and its online services customers concerning the protection of personal data.

The GitHub DPA is available here: https://github.com/customer-terms/github-data-protection-agreement

### Which customers does the GitHub DPA apply to?

The GitHub DPA will apply to a customer if the GitHub Product Specific Terms/Microsoft Product Terms for GitHub Offerings for the products the customer is purchasing incorporates the DPA.

The GitHub DPA does not apply to GitHub Enterprise Server customers where all data is stored on premises at the customer's site.

The Microsoft DPA does not cover GitHub products. Even if a customer has a DPA in place with Microsoft, the GitHub DPA will apply in respect of the GitHub online services.

### What is the difference between the GitHub Data Protection Agreement and the GitHub General Privacy Statement?

The GitHub Data Protection Agreement applies where GitHub acts as a processor of the customer's personal data.

The GitHub General Privacy Statement applies where GitHub acts as a controller of the customer's personal data.

### Why do GitHub and Microsoft have different DPAs?

Although GitHub is owned by Microsoft, GitHub is a different legal entity, it has different infrastructure, different personnel and different policies. Therefore, we have our own separate DPA to be legally compliant with privacy laws.

### Can we agree to use a customer's DPA?

No. We cannot agree to use a customer's DPA. GitHub's DPA was intentionally written to reflect actual GitHub practice.

### Can a customer negotiate the GitHub DPA?

We can consider customer redlines to the GitHub DPA if the customer reaches the minimum threshold of $500k ARR. The threshold is set at $500k because DPA negotiations involve multiple different teams and require significant time and effort.

### What if a customer wants to sign the GitHub DPA?

This question about signing the DPA comes up occasionally because of GDPR Article 28 which requires that a data controller have a DPA with its processor that is "a contract or other legal act under Union or Member State law, that is binding on the processor" and "shall be in writing".

Some customers assume that means the actual DPA document must be signed but that is not necessary.

On the GitHub side, the customer has typically signed the Order Form or other document that incorporates the GitHub Customer Agreement, including the GitHub Product Specific Terms and the GitHub DPA.

It is the Order Form or Program Signature Form (or whatever document the customer signed) that is the binding signature. In short, GitHub's view is that the customer has already provided a signature in satisfaction of GDPR Article 28.

## Copilot Questions

### Where can I find information on the GitHub Copilot contract stack?

You can find information on how the GitHub Copilot contract stack works for GitHub direct deals and Microsoft co-sells in the Copilot Contract Stack Quick Guide on SharePoint.

### My customer has questions about the GitHub Copilot indemnity.

The Copilot Contract Stack Quick Guide contains information on how the GitHub Copilot indemnity works under the GitHub contract stack and the Microsoft contract stack.

"Indemnity" is a contractual term that requires one contracting party to defend and/or compensate the other party for certain damages caused in connection with the contract. In GitHub contracts, this comes up most frequently with customers concerned that GitHub Copilot will infringe upon the intellectual property rights of third parties (such as sending code snippets belonging to someone else). While this risk is incredibly low in real life, the contractual indemnity is our way of easing these customer concerns.

## Other Common Questions

### Can we agree to a Customer's Code of Conduct/Supplier Code?

We cannot agree to individual Customer Codes of Conduct/Supplier Codes. GitHub operates on a one-to-many basis and it is not possible to agree to bespoke customer requirements.

As a Microsoft company, we adhere to the Microsoft Standards of Business Conduct (aka the Trust Code) which is an extensive business conduct policy. Please direct the customer to this link where they can download the Microsoft Trust Code: https://www.microsoft.com/en-us/legal/compliance/sbc/standards

### My customer has asked GitHub to complete a questionnaire. What do I do next?

Please follow the Customer Security Trust process: Working with Security Questionnaires at https://github.com/github/solutions-engineering/blob/master/guides/internal-tools/working_with_security_questionnaires.md

### Where can I find a document showing that GitHub is owned by Microsoft?

Use the .terms me Hubot command in Hubot-enabled Slack channels to bring up the Microsoft - GitHub ownership certificate. Please ensure there is an NDA in place before sharing.

### Where can I find information about GitHub's insurance policies?

Use the .terms me Hubot command in Hubot-enabled Slack channels to bring up GitHub's insurance documents. Please ensure there is an NDA in place before sharing.

### Where can I find company details & finance information about GitHub Inc?

You can find more information in the Vendor Form Information document: https://github.com/github/financeteam/blob/master/vendor-form-information.md

### Trade Screen Status showing "Hit in Review"

Every time a new customer is added to Salesforce, or every time there's a change to an address on a customer account, the data runs through a trade screen to make sure we're allowed to do business with this company. Most of the time, this screen clears on its own within 1-2 days. If the lock still hasn't cleared after 2 days, please let #legal know.
